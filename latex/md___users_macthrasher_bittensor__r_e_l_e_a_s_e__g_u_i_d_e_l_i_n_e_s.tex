The release manager in charge can release a Bittensor version using two scripts\+:
\begin{DoxyItemize}
\item \href{./scripts/release/versioning.sh}{\texttt{ ./scripts/release/versioning.sh}}
\item \href{./scripts/release/release.sh}{\texttt{ ./scripts/release/release.sh}}
\end{DoxyItemize}

The release manager will need the right permissions for\+:
\begin{DoxyItemize}
\item github.\+com
\item pypi.\+org
\item hub.\+docker.\+com
\end{DoxyItemize}

If you are new in this role, ask for the proper setup you need to run this process manually.\hypertarget{md___users_macthrasher_bittensor__r_e_l_e_a_s_e__g_u_i_d_e_l_i_n_e_s_autotoc_md89}{}\doxysection{Process of release}\label{md___users_macthrasher_bittensor__r_e_l_e_a_s_e__g_u_i_d_e_l_i_n_e_s_autotoc_md89}

\begin{DoxyEnumerate}
\item Create a branch called {\ttfamily release/\+VERSION}, having VERSION with the version to release.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item Within the release branch; Update the version using the versioning script\+:
\begin{DoxyEnumerate}
\item {\ttfamily ./scripts/release/versioning.sh -\/-\/update UPDATE\+\_\+\+TYPE}, UPDATE\+\_\+\+TYPE could be major, minor or patch.
\end{DoxyEnumerate}
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item Test the release branch and verify that it meets the requirements.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item After merging the release branch; Run the release script
\end{DoxyEnumerate}\hypertarget{md___users_macthrasher_bittensor__r_e_l_e_a_s_e__g_u_i_d_e_l_i_n_e_s_autotoc_md90}{}\doxysection{Versioning script usage}\label{md___users_macthrasher_bittensor__r_e_l_e_a_s_e__g_u_i_d_e_l_i_n_e_s_autotoc_md90}
Options\+:
\begin{DoxyItemize}
\item -\/U, --update\+: type of update. It could be major, minor, patch or rc (release candidate).
\item -\/A, --apply\+: This specify to apply the release. Without this the versioning will just show a dry run with no changes.
\end{DoxyItemize}\hypertarget{md___users_macthrasher_bittensor__r_e_l_e_a_s_e__g_u_i_d_e_l_i_n_e_s_autotoc_md91}{}\doxysection{Release script usage}\label{md___users_macthrasher_bittensor__r_e_l_e_a_s_e__g_u_i_d_e_l_i_n_e_s_autotoc_md91}
Options\+:
\begin{DoxyItemize}
\item -\/A, --apply\+: This specify to apply the release. Without this the release will just show a dry run with no changes.
\item -\/T,--github-\/token\+: A github personal access token to interact with the Github API.
\end{DoxyItemize}\hypertarget{md___users_macthrasher_bittensor__r_e_l_e_a_s_e__g_u_i_d_e_l_i_n_e_s_autotoc_md92}{}\doxysubsection{Github token}\label{md___users_macthrasher_bittensor__r_e_l_e_a_s_e__g_u_i_d_e_l_i_n_e_s_autotoc_md92}
Since you need to use a secret when releasing bittensor (github personal access token), I encourage you to use \href{https://www.passwordstore.org/}{\texttt{ pass}} or a similar tool that allows you to store the secret safely and not expose it in the history of the machine you use.

So you can have\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{GITHUB\_ACCESS\_TOKEN=\$(pass\ github/your\_personal\_token\_with\_permisions)}

\end{DoxyCode}


or 
\begin{DoxyCode}{0}
\DoxyCodeLine{GITHUB\_ACCESS\_TOKEN=\$(whatever\ you\ need\ to\ get\ the\ token\ safely)}

\end{DoxyCode}
